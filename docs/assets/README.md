# README assets

Screenshots/GIFs embedded in the top-level [README.md](../../README.md). Drop the
files here using the exact names below so the README's image links resolve.

| File | Priority | What to capture |
|---|---|---|
| `dashboard-watchlist.png` | ⭐ hero | The home page — tiered S/A/B/C company cards with logos + "Expected: <month>". |
| `fit-score.png` | ⭐ high | The **fit** tab with a scored result: the 0–100, component bars, matched/missing skills, tips. |
| `telegram-alert.png` | ⭐ high | A real Telegram alert on your phone (a 🔔 watchlist or 🆕 firehose message). Blur anything personal. |
| `company-detail.png` | medium | A company detail page showing the "expected to open" seasonality chart + open roles. |
| `sentiment-card.png` | medium | A generated company sentiment report card (close-up). |
| `firehose.png` | optional | The firehose feed with "posted <date>" rows. |

**Capture tips**
- PNG, dark theme, full content width (≈1400px wide reads well on GitHub). Keep widths consistent.
- Pick data-rich examples: a watchlist company with a real expected-open month; a fit score that isn't 0; a sentiment card you've generated first.
- A short screen-recording → GIF of an alert firing (or the fit flow) is great if easy; name it `demo.gif`.
